## Design Context

### Users
Backend and full-stack developers who document APIs, test HTTP endpoints, and query databases as part of their daily workflow. They value speed, local-first tools, and plain-text formats (markdown, git). They work during the day in dark IDEs and terminals, often switching between Postman, DBeaver, and documentation tools. httui replaces that fragmented workflow with one editor.

### Brand Personality
**Pragmatic, capable, sharp.** Not flashy or playful — confident and functional. The tool does real work; the brand should feel like a well-made instrument, not a toy. Think: the confidence of a CLI tool with the polish of a native app.

### Aesthetic Direction
- **Theme:** Dark mode (matches the app and developer context)
- **Tone:** Technical but approachable. Dense information presented cleanly. Let the product screenshots do the talking.
- **References:** HTTPie (bold dark theme, strong accent color, community feel, large confident typography), Bruno (clean structure, developer credibility, feature cards)
- **Anti-references:** Generic SaaS landing pages with gradient heroes, glassmorphism, floating 3D elements. Postman's corporate/enterprise feel. Overly minimal pages that don't show the product.
- **Key differentiator:** The page should feel like looking at a real editor — the mock blocks ARE the feature demos, not abstract illustrations or icons.

### Design Principles
1. **Show, don't describe** — Every feature section demonstrates the actual UI, not an icon or illustration. The product is the hero.
2. **Dense but clear** — Developers scan fast. Pack information tightly but with clear hierarchy. No filler sections.
3. **Earned confidence** — Bold typography and strong contrast. No hedging language, no "try it maybe." The tool works; say so directly.
4. **Dark and sharp** — High contrast dark theme with a single strong accent. No gradients, no glows, no glass. Clean edges, solid colors.
5. **Local-first identity** — Emphasize ownership, plain files, no cloud. This is a core value, not a footnote.
